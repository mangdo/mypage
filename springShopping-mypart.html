담당 기능 상세 설명

1) ERD 설계

: 전체 프로젝트 중 제가 설계한 부분만 발췌하였습니다.

2) 로그인

: Spring Security를 사용하여 로그인을 할 수 있습니다.
: Spring Security Handler를 사용하여 로그인 성공, 실패, 권한에 따른 접근 제한을 처리합니다.
: 일반 회원 계정(아이디 : user1, 비밀번호: a1234)으로 로그인하실 수 있습니다.

3) 제품의 CRUD

: 전체 구조

: 제품 등록 페이지

: 제품의 이미지 등록 상세

: 제품 조회 페이지

4) 리뷰와 답글 CRUD

: REST API로 설게하였습니다.
: json을 반환하는 RestController를 사용하였고, View단에서 JQuery의 AJAX로 데이터를 주고 받습니다.

: 리뷰와 답글의 전체 흐름

5) 카카오페이 결제

: 카카오 REST API를 이용하여 카카오페이로 결제를 진행합니다.

6) 문의 메일 전송

: JavaMailSender을 이용하여 메일을 전송합니다.
: 메일은 단순한 텍스트만 담고 있기때문에 SimpleMailMessage를 이용합니다.

7) 공지의 CRUD

: 관리자는 공지를 등록, 수정, 삭제할 수 있으며 누구나 조회할 수 있습니다.
: 공지는 이미지를 등록시킬 수 있습니다.

8) AWS 배포

: AWS EC2 (Amazon linux 2), AWS RDS (MySQL), AWS S3를 사용하여 배포하였습니다.
: 이에 대한 자세한 회고는 블로그 포스팅[AWS 배포 회고]를 통해 확인하실 수 있습니다.
